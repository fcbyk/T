# demo6 分支选择 `|`
`|` 是正则里的 **「或匹配」符号**，作用就是：**满足左边 或者 右边 任意一个规则，都算匹配成功**。

## 一、核心定义
```
A|B
```
意思：**匹配 A ，或者匹配 B**，二选一，谁先出现匹配谁。

- 左边：一个正则规则
- 右边：另一个正则规则
- 匹配规则：**从左往右依次检查，找到第一个符合的就停止**



## 二、最简单的例子
### 例1：匹配 cat 或 dog
正则：`cat|dog`

能匹配：
- `cat` ✅
- `dog` ✅

不能匹配：
- `bird` ❌
- `cats` ❌



### 例2：匹配数字 1 或 2
正则：`1|2`

能匹配：
- `1` ✅
- `2` ✅



## 三、可以无限叠加多个分支
不是只能两个，**可以写无数个**：
```
A|B|C|D|E
```
匹配：A 或 B 或 C 或 D 或 E。

例子：匹配 苹果/香蕉/橙子
```
苹果|香蕉|橙子
```



## 四、优先级问题
`|` 的**优先级非常低**，比普通字符、量词（*+?）都低。

### 用括号 `()` 限定范围
括号可以**把分支包起来**，表示「这一整块是一个选项」：
```
(abc)|(def)
```
或者简写：
```
(abc|def)
```



## 五、实战常用例子
### 1. 匹配 http 或 https
```
https?|http
```
更标准写法：
```
http(s)?
```

### 2. 匹配 .com 或 .cn 或 .net
```
\.com|\.cn|\.net
```

### 3. 匹配男/女
```
男|女
```

### 4. 匹配 1~9 的数字
```
1|2|3|4|5|6|7|8|9
```



## 六、匹配顺序：从左到右，找到即停止
`a|ab|abc`

匹配字符串 `abc`：
只会匹配第一个 `a`，不会匹配更长的 `ab` 或 `abc`。

✅ 规则：**谁写在左边，优先匹配谁**。



## 七、总结成 3 句口诀
1. `A|B` = 匹配 A 或 匹配 B
2. 多个分支：`A|B|C`
3. 想限定范围必须加 **括号**：`(A|B)`


### 总结
- `|` = **或**
- 作用：多个规则任选其一匹配
- 优先级低，**一定要用括号控制范围**
- 匹配顺序：**从左往右，找到第一个就结束**
