# demo5 分组、捕获、反向引用

## 一、先搞懂：什么是分组？
**分组**就是用小括号 `()` 把**一段正则表达式包起来**，当成一个**整体单元**使用。

### 作用
1. 把多个字符当成**一个整体**，给它们加量词（`* + ? {n}`）
2. 为后续**捕获、反向引用**做准备

### 例子
```
正则：(ab)+
匹配：ab abab ababab
```
- 没有括号：`ab+` → 匹配 `a` 后面跟着**一个或多个 b**（ab、abb、abbb）
- 有括号：`(ab)+` → 把 `ab` 当成**一个整体**，匹配**一次或多次**

## 二、什么是捕获？
**捕获** = 分组 + 把分组匹配到的内容**保存下来**。

只要你用了 `()` 分组，正则引擎就会**自动把这个分组匹配到的内容存起来**，这就叫**捕获分组**。

### 特点
1. 用 `()` 定义
2. 自动编号：`$1`、`$2`、`$3`...（从左到右数左括号）
3. 可以在后续代码/替换中取出这些内容

### 例子：提取手机号的运营商号段
```
文本：13812345678
正则：(13\d)(\d{4})(\d{4})
```
匹配后会捕获 3 组内容：
- 第 1 组 `$1` → `138`
- 第 2 组 `$2` → `1234`
- 第 3 组 `$3` → `5678`

### 不想捕获？用非捕获分组
如果你只是想分组，**不想保存内容**，用 `(?:)`
```
(?:ab)+  → 只分组，不捕获，不占编号
```

## 三、什么是反向引用？
**反向引用** = 在**同一个正则表达式里**，**重复使用前面捕获到的分组内容**。

语法：
- 正则内部：`\1`、`\2`、`\3`...
- 对应第 1、2、3 个捕获分组

### 核心作用
匹配**重复出现的相同内容**

## 四、最经典例子：匹配连续重复的字符
### 需求
匹配像 `aa`、`bb`、`11`、`@@` 这样**两个相同字符连在一起**的字符串

### 正则
```
(.)\1
```

### 拆解
1. `(.)`：**分组+捕获**
   - `.` 匹配任意一个字符
   - `()` 把它捕获成 `$1` / `\1`
2. `\1`：**反向引用**
   - 引用第 1 个分组匹配到的内容
   - 意思是：这里必须和刚才捕获的字符**一模一样**

### 匹配结果
- `aa` ✅ 匹配
- `11` ✅ 匹配
- `ab` ❌ 不匹配
- `@@` ✅ 匹配

## 五、进阶例子：匹配 HTML 标签
### 需求
匹配正确的闭合标签，比如 `<div>...</div>`、`<p>...</p>`

### 正则
```
<([a-z]+)>[^<]*</\1>
```

### 拆解
1. `<([a-z]+)>`：捕获标签名 → `$1`
2. `[^<]*`：匹配中间内容
3. `</\1>`：**反向引用**前面的标签名，确保闭合标签和开始标签一致

### 效果
- `<div>hello</div>` ✅ 匹配
- `<p>test</p>` ✅ 匹配
- `<div>test</p>` ❌ 不匹配（标签不统一）

## 六、快速总结（必背）
| 概念         | 符号       | 作用                                   |
|--------------|------------|----------------------------------------|
| 分组         | `()`       | 把多个字符当成**一个整体**             |
| 捕获         | `()`       | 把分组内容**存起来**，编号 `$1、$2`    |
| 非捕获分组   | `(?:)`     | 只分组，**不保存**内容                 |
| 反向引用     | `\1、\2`   | 在正则**内部重复使用**捕获的内容       |

### 一句话记忆
**括号分组，自动捕获，反斜杠加数字就是引用！**

### 总结
1. **分组**：`()` 把一段正则当成整体
2. **捕获**：分组后自动保存内容，用 `$1` 取
3. **反向引用**：`\1` 在正则内部重复使用前面捕获的内容
4. 最常用场景：**去重、匹配重复字符、匹配闭合标签**
