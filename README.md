# demo30

## 一、什么是 Vue 组件
一句话：
**Vue 组件就是把页面拆成一个个独立、可复用的“小模块”。**

就像搭乐高：
- 一个页面 = 一整座乐高城堡
- 一个组件 = 一块乐高积木（按钮、导航栏、卡片、弹窗...）

特点：
1. **可复用**：写一次，到处用
2. **独立封闭**：自己的代码、样式、数据互不干扰
3. **可组合**：组件里还能套组件
4. **易维护**：改一个地方，所有用到的地方自动更新

## 二、组件底层

**Vue 组件本质就是一个普通的 JS 对象！**

不管是 Vue2 还是 Vue3，底层长这样：
```js
const MyComponent = {
  name: 'MyComponent',
  data() {},
  props: {},
  methods: {},
  template: `<div>字符串模板</div>`
  render() {}  // 最关键：模板最终会编译成 render 函数
}
```

## 三、模板与渲染函数

SFC写的：
```vue
<template>
  <div>hello</div>
</template>
```

**底层会被编译成：**
```js
render(){
  return h('div', 'hello')
}
```

总结：
- **模板只是语法糖**
- **真正运行的是 render 函数**

## 四、完整底层链路：从 SFC 到页面
> **组件对象本身不生成 VNode，是运行时调用它的 render 函数生成 VNode。**

1. **SFC（.vue）** 只是给人写的文件，浏览器不认
2. **Vite 编译** → 把 SFC 变成 **组件配置对象（JS）**
3. **Vue 应用实例（createApp）** 是总指挥
4. **Vue 运行时** 负责：
   - 加载组件对象
   - 执行渲染函数 → 生成 **虚拟 DOM（VNode）**
   - 把虚拟 DOM → 变成 **真实 DOM**
   - 挂载到页面

### 一些问题

1. 生成组件配置对象后，**谁来生成虚拟 DOM？**
    - **Vue 运行时（runtime-core）根据组件对象 + 渲染函数生成 VNode**
    - 你可以理解：
        - **组件对象 = 图纸**
        - **Vue 运行时 = 施工队**
        - **VNode = 建筑模型（虚拟DOM）**
        - **真实 DOM = 盖好的房子**
    - 不是应用实例，应用实例只是管理者

2. 是 Vue 应用实例在干活吗？
   -  **应用实例是总指挥，干活的是 Vue 底层运行时。**

3. 生成配置对象后，直接生成虚拟 DOM 吗？
   - **不是直接**
   - 流程是：配置对象 → 创建组件实例 → 运行 render 函数 → 得到 VNode

### 总结

```
你写 SFC
   ↓ (Vite 编译)
组件配置对象（JS）
   ↓ (createApp 创建应用实例)
Vue 应用实例（总指挥）
   ↓ (运行时初始化)
组件实例
   ↓ (运行时调用 render)
虚拟 DOM（VNode）
   ↓ (渲染器 patch)
真实 DOM → 显示在页面
```
