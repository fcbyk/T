# demo7 环视（断言）
正则里的**环视（Lookaround）**，也叫**断言**，是正则最强大、最容易被忽略的功能——**它不匹配任何字符，只匹配「位置」**，就像给字符串里的「空隙」加了判断条件。

简单说：**环视只看位置，不占字符，匹配成功后不消耗字符**。

## 一、先搞懂核心：什么是「位置」？
字符串里每个字符之间、开头、结尾都是**位置**：
```
↑  h  e  l  l  o  ↑
开头                 结尾
```
环视就是判断：**这个位置，是否满足我设定的条件**。



## 二、4 种环视（断言）全分类
正则只有 **4 种标准环视**，分成两组，非常好记：

| 类型 | 符号 | 中文名字 | 一句话理解 |
|------|------|----------|------------|
| 正向先行断言 | `(?=...)` | 后面是 | 这个位置**后面**必须跟着 ... |
| 负向先行断言 | `(?!...)` | 后面不是 | 这个位置**后面**不能跟着 ... |
| 正向后行断言 | `(?<=...)` | 前面是 | 这个位置**前面**必须是 ... |
| 负向后行断言 | `(?<!...)` | 前面不是 | 这个位置**前面**不能是 ... |

### 记忆口诀
- `=` 是**肯定**（满足条件）
- `!` 是**否定**（不满足条件）
- `<` 是**往回看（前面）**，没有 `<` 是**往前看（后面）**



## 三、逐个拆解 + 实战例子
我用最简单的例子，让你一看就懂。

### 1. 正向先行断言：`(?=条件)`
**作用**：匹配一个**位置**，这个位置**后面必须是指定内容**。

#### 例子：匹配 `hello` 里，后面跟着 `l` 的位置
```regex
(?=l)
```
在 `hello` 中：
- 第 2 个字符 `e` 和第 3 个字符 `l` 之间的位置 ✅
- 第 3 个字符 `l` 和第 4 个字符 `l` 之间的位置 ✅

#### 实战：给数字加千分位（最经典用法）
需求：把 `1234567` → `1,234,567`
```regex
(?=(\d{3})+$)
```
含义：**从后往前数，每 3 个数字的前面位置**，插入逗号。



### 2. 负向先行断言：`(?!条件)`
**作用**：匹配一个**位置**，这个位置**后面不能是指定内容**。

#### 例子：匹配后面**不是** `l` 的位置
```regex
(?!l)
```

#### 实战：禁止以数字开头
```regex
^(?!\d)
```
含义：字符串开头位置，后面**不能是数字**。



### 3. 正向后行断言：`(?<=条件)`
**作用**：匹配一个**位置**，这个位置**前面必须是指定内容**。

#### 例子：匹配前面是 `e` 的位置
```regex
(?<=e)
```
在 `hello` 中：`e` 和 `l` 之间的位置 ✅

#### 实战：提取 @ 后面的用户名
文本：`我的账号是 @abc123`
```regex
(?<=@)\w+
```
含义：**@ 后面的所有字母数字**，直接匹配出 `abc123`。



### 4. 负向后行断言：`(?<!条件)`
**作用**：匹配一个**位置**，这个位置**前面不能是指定内容**。

#### 例子：匹配前面**不是** `h` 的位置
```regex
(?<!h)
```

#### 实战：匹配独立的 cat（不是 catty 里的 cat）
```regex
(?<!\w)cat(?!\w)
```



## 四、最关键的特性：环视「不占字符」
这是新手最容易踩坑的点：
### ❌ 错误理解
以为 `a(?=b)` 会匹配 `ab` 两个字符。
### ✅ 正确理解
`a(?=b)` **只匹配字符 a**，条件是 a 后面必须跟着 b。
- 匹配结果：`a`
- 不消耗 b，b 还留在字符串里

**例子**：
字符串：`abc`
正则：`a(?=b)`
匹配到：**`a`**（b 只是判断条件，不被匹配）



## 五、超级实用的实战场景
### 1. 密码强度校验（必学！）
要求：8-16位，必须包含**大写、小写、数字**
```regex
^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,16}$
```
拆解：
- `(?=.*[a-z])`：字符串里**必须有小写字母**
- `(?=.*[A-Z])`：必须有大写
- `(?=.*\d)`：必须有数字
- 所有条件同时满足！

### 2. 给手机号中间 4 位打码
`13812345678` → `138****5678`
```regex
(?<=\d{3})\d{4}(?=\d{4})
```
直接替换成 `****` 即可。

### 3. 匹配不包含某个词的字符串
匹配不包含 `error` 的行：
```regex
^(?!.*error).*$
```



## 六、快速总结（背会这 4 个就够了）
1. `(?=A)` → 位置**后面是 A**
2. `(?!A)` → 位置**后面不是 A**
3. `(?<=A)` → 位置**前面是 A**
4. `(?<!A)` → 位置**前面不是 A**

核心：**只看位置，不匹配字符，不消耗字符**。



### 总结
- 环视=断言=匹配**位置**，不占字符
- 4 种固定写法，`=`肯定、`!`否定、`<`看前面
- 最常用：密码校验、提取内容、格式化数字、过滤文本
